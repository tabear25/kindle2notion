## 全体像（なぜ手作業が要るのか）

このスクリプトは Google の「サービスアカウント」という機械用アカウントで Drive を操作します。
サービスアカウントには **Drive の保存容量が 0 バイトしかない** ため、
**新しいファイルを自分で作れません**（フォルダは作れる）。

そのため、

- **空のスプレッドシート 50個を作るのは人間（自分）の手作業**
- **その50個に中身を書き込むのはスクリプト**

という分担になります。ファイル名・個数は固定なので、50個を一度用意すれば
それ以降ずっと使い回せます。

## ステップ2：dry-run（下見）で「作るべきファイル名」を確認する

### dry-run とは何か

`--apply` を付けずにスクリプトを実行すると **dry-run（ドライラン＝予行演習）モード** になります。
このモードでは **Drive には一切書き込みません**。やるのは次のことだけ：

- 既存の 49 ボリュームを読む（現在の挙動。旧マスタは読みません）
- 「50個のうち、どれが既に存在し、どれがまだ無いか」をチェックする
- 「これから何をするつもりか」を画面に表示する

つまり **安全な確認用** です。何度実行しても Drive は変わりません。

### 実行コマンド

`kindle2notion` フォルダ（このプロジェクトのフォルダ）で、ターミナルを開いて実行：

```bash
py -3 -m scripts.split_per_book --parent-folder FOLDER_ID
```

`FOLDER_ID` の部分はステップ1で控えた実際のIDに置き換える。
（`py -3` が動かない場合は `python` でも可）

### 出力の読み方

こんな表示が出ます：

```
[master] 49 books, 812 highlights
[layout] 49 volumes + 1 index = 50 files, prefix 'k2n'
...
  [create ] k2n_index      (49 books)
  [create ] k2n_vol_01     (2 books, 31 highlights)
  [create ] k2n_vol_02     (0 books)
  ...
[summary] update=0  missing=50
```

- `[create ]` … そのファイルがまだ Drive に無い、という意味（dry-run なのでまだ作らない）
- `[update ]` … そのファイルは既にある、という意味
- 末尾に **作るべきファイル名の一覧** がまとめて表示される
- 最初は全部 `[create ]`／`missing=50` になるはず（まだ50個作っていないので正常）

この時点で **作るべき50個のファイル名が確定** します。名前は固定で、必ず次の50個です：

```
k2n_index
k2n_vol_01  k2n_vol_02  k2n_vol_03  ... k2n_vol_49
```

（`k2n_vol_01`〜`k2n_vol_49` の49個 ＋ `k2n_index` の1個）

## ステップ3：空のスプレッドシートを50個作る

ステップ1で作った `notebooklm` サブフォルダの中に、
**空の Google スプレッドシートを50個** 作ります。

### 重要なルール

- ファイル名は上の50個と **完全一致** させる（大文字小文字・アンダースコア・桁数まで）
  - 例：`k2n_vol_07` は OK、`k2n_vol_7`（ゼロ無し）や `k2n-vol-07` は **NG**
- **拡張子は付けない**（`.xlsx` などは不要。Google スプレッドシートはそのままでよい）
- 中身は空のままでよい（中身はスクリプトが書き込む）
- 必ず `notebooklm` フォルダの **中** に作る

スクリプトはこの「ファイル名」を頼りに書き込み先を探します。名前が1文字でも違うと
そのファイルは見つけてもらえず `[missing]` のままになります。

### 手作業で作る場合

`notebooklm` フォルダを開いた状態で「新規 → Google スプレッドシート」を50回。
作るたびに名前を上のリストの通りに変更する。手間ですが **1回きり** です。

### （おすすめ）Apps Script で50個まとめて作る

50回の手作業がつらい場合、Google Apps Script で一括作成できます。

1. <https://script.google.com> を開く →「新しいプロジェクト」
2. コードを全部消して、下を貼り付ける
3. `フォルダIDをここに貼る` を、`notebooklm` フォルダのID（`notebooklm` を
   開いたときの URL の `/folders/●●●` 部分）に置き換える
4. 上部の ▷「実行」を押す（初回は権限の許可を求められるので承認する）

```javascript
function createNotebookLMSheets() {
  var folderId = 'フォルダIDをここに貼る'; // ← notebooklm フォルダのID
  var folder = DriveApp.getFolderById(folderId);

  var names = ['k2n_index'];
  for (var i = 1; i <= 49; i++) {
    names.push('k2n_vol_' + ('0' + i).slice(-2)); // k2n_vol_01 .. k2n_vol_49
  }

  names.forEach(function (name) {
    if (folder.getFilesByName(name).hasNext()) return; // 既にあればスキップ
    var ss = SpreadsheetApp.create(name);
    DriveApp.getFileById(ss.getId()).moveTo(folder);
  });
}
```

これで `notebooklm` フォルダの中に50個の空スプレッドシートが一気にできます。
作成者は自分なので容量の問題は起きません。

### 確認

dry-run（ステップ2のコマンド）をもう一度実行し、`[create ]` だった行が
`[update ]` に変わり、`missing=0` になっていれば50個そろっています。

## ステップ4：apply（本番書き込み）を実行する

### apply とは何か

`--apply` を付けて実行すると **本番モード** になり、**実際に Drive へ書き込みます**。
現在の挙動では：

- 既存の 49 ボリュームを読み、`k2n_index`（索引）を作り直して書き込む
- ボリューム本体は上書きしません（索引の再構築だけ）

これは索引が壊れた／古くなったときの保守用です。**通常の運用では実行不要** で、
ハイライトは取得・手動追加のたびに自動でボリューム＋索引へ反映されます。

> 旧マスタから 50 ファイルを丸ごと作り直す（＝以前のステップ4の挙動）には
> `--from-master` を付けます。ただしこれは取得済みの新ハイライトを上書きで
> 失う恐れがあるため、初回の一括移行などレガシー用途に限ります。
> 書き込み先は **ステップ3で用意した50個の既存ファイルだけ**。新しいファイルは作りません。

### 実行コマンド

```bash
py -3 -m scripts.split_per_book --apply --parent-folder FOLDER_ID
```

dry-run のコマンドに **`--apply` を足しただけ**。`FOLDER_ID` は同じものを使う。

### 成功の確認

最後に次が出れば成功です：

```
[summary] update=50  missing=0
```

- `update=50` … 50個すべてに書き込めた
- `missing=0` … 見つからなかったファイルは無し

もし `missing` が 0 でない場合、その名前のファイルがまだ無い／名前が違うので、
ステップ3に戻って不足分を作り、もう一度 apply する。

### 2回目以降（新しい本が増えたとき）

**手作業は不要になりました。** Kindle 取得（`main.py` / Web）や手動追加
（`add_manual_highlights --apply`）を実行すると、その場で該当ボリューム＋索引へ
自動反映されます。**ファイルを新しく作る必要もありません**（50個固定なので）。
`NOTEBOOKLM_PARENT_FOLDER_ID` が設定されていることだけ確認してください。
